Layer 1 Dynamic Market Data Engine (yfinance + MCP) ▾
Data flow

Files

Layer 2 News Intelligence Agent (Why did it move?) ▾
Data flow
RSS / NewsAPI MCP → NLP sentiment (VADER + transformer) → news_articles table → LLM "why did it move" summary
Files
agents/news_intel_agent.py
Polls RSS feeds (Yahoo Finance, Reuters, Bloomberg RSS) every 15min per symbol. Runs VADER + optional HuggingFace finbert. Stores headline, url, sentiment_score, symbol, published_at into news_articles table.
agents/move_explainer.py
New agent. On each market_update event, reads the last 6h of news for that symbol. Calls local LLM with a prompt asking "Given these headlines and the price moved X%, what is the primary cause?" Returns a JSON: {reason, sentiment, confidence}. Persists to agent_insights.
gateway/server.py
Add GET /news/{symbol}?hours=24 — returns articles + move explanation from agent_insights. Used by frontend to show "why did AAPL drop 3%?" card.
LLM system prompt inside move_explainer.py
You are a financial analyst AI. You will receive: - SYMBOL: the ticker/coin slug - PRICE_CHANGE: percentage change in last {window}h - HEADLINES: list of recent news headlines with timestamps Your task: 1. Identify the PRIMARY reason for the price movement. 2. Classify sentiment: BULLISH | BEARISH | NEUTRAL 3. Rate your confidence 0–100. 4. Cite the 1–2 most relevant headlines. Respond ONLY in JSON: {"reason": "...", "sentiment": "BULLISH|BEARISH|NEUTRAL", "confidence": 85, "key_headlines": ["...", "..."]}
Layer 3 LLM Memory & DB Reader (Ask anything about history) ▾
Data flow
User query → Qdrant vector search on agent_insights → SQLite raw context → LLM synthesizes answer
Files
agents/market_rag.py
On every agent_insight write, embed the insight text with a sentence-transformer and upsert into Qdrant collection "market_memory". On query, embed the question, retrieve top-5 chunks, fetch matching OHLCV rows from SQLite, pass both to LLM.
gateway/server.py
Add POST /ask endpoint: {symbol?, question}. Routes to market_rag.py which does vector retrieval + LLM answer. Returns streaming SSE response so UI can show tokens arriving live.
LLM system prompt inside market_rag.py
You are AITradra, an institutional market intelligence AI with access to: - Historical OHLCV data stored in your database - Past agent analysis and insights - Recent news and sentiment records Answer the user's question using ONLY the provided context below. If the answer is not in the context, say "I don't have enough data on that." Always cite the date/source of data you reference. Be concise. Structure: [Direct answer] → [Supporting data] → [Caveat if any]. CONTEXT: {retrieved_chunks} RAW DATA: {ohlcv_rows} USER QUESTION: {question}
Layer 4 Mythic Consensus Pipeline (Real signal scoring) ▾
Data flow
Technical score (40%) News sentiment (40%) Social/macro (20%) → Mythic consensus JSON → research_suggestions table
Files
agents/mythic_orchestrator.py
Reads latest technical score from TechnicalSpecialist, sentiment score from NewsIntelAgent, and macro score from MacroSentryAgent. Applies 40/40/20 weighted average. If volume > 1.5x avg applies 1.2× multiplier. Writes final conviction score + direction to research_suggestions.
agents/technical_specialist.py
Reads OHLCV from SQLite. Computes SMA20/50 crossover, RSI(14), MACD signal, volume ratio. Returns a score -1.0 to +1.0. Uses pandas-ta. Writes to agent_insights.
agents/accuracy_store.py
Background job runs every 24h. For each research_suggestion older than 24h, fetches actual close vs predicted direction. Computes binary accuracy. Writes to agent_health table. Feeds influence weight back to orchestrator.
Mythic orchestrator final LLM call prompt
You are the Mythic Orchestrator for AITradra. SYMBOL: {symbol} TECHNICAL_SCORE: {tech_score} (range -1 to +1) NEWS_SENTIMENT_SCORE: {news_score} (range -1 to +1) MACRO_SCORE: {macro_score} (range -1 to +1) WEIGHTED_CONSENSUS: {final_score} PRICE_CHANGE_24H: {pct_change}% MOVE_REASON: {move_reason} Produce the final institutional signal. Respond ONLY in JSON: { "direction": "LONG | SHORT | NEUTRAL", "conviction": 0-100, "entry_zone": "price range string", "stop_loss": "price", "target": "price", "rationale": "2-3 sentence summary", "risk_level": "LOW | MEDIUM | HIGH" }
Layer 5 MCP Open Source Integrations ▾
MCP servers to wire up
mcp/crypto_gateway.py
Wraps coingecko-mcp (open source). Fetches price, market cap, 24h volume, 7d change for BTC/ETH/SOL/any slug. Run: npx @llmindset/mcp-coingecko
mcp/news_mcp.py
Wraps newsapi-mcp or fetch-mcp for RSS. Fetches top 10 headlines per symbol query. Run: npx @modelcontextprotocol/server-fetch
mcp/memory_mcp.py
Wraps @modelcontextprotocol/server-memory (KV memory MCP). Each agent writes its last output as a named entity. Orchestrator reads all entities to build context without hitting SQLite every call.
mcp/brave_search.py
Optional: wraps @modelcontextprotocol/server-brave-search. Used by DeepResearch agent for real-time macro news not in RSS (Fed statements, earnings releases).
